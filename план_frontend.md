Техническая спецификация: Frontend

Дополнение к основному ТЗ (`online_kutibxona_v2_spec.md`). Единый UI-слой (`dashboard/`) для всех ролей — не разделять на отдельные приложения.

### Палитра (dark theme)

Тёмная тема — **контрастная, близкая к чёрной** (это операционная CRM-панель с таблицами и данными, а не читалка сплошного текста — приоритет чёткости данных, не смягчённый комфорт для долгого чтения прозы).

### Типографика

- **Display/заголовки:** шрифт с чёткими геометричными формами (например, Manrope или Sora) — используется в заголовках экранов, крупных цифрах статистики, названиях достижений.
- **Основной текст:** нейтральный, максимально читаемый шрифт для таблиц/форм (например, Inter или системный стек) — большая часть интерфейса это данные, не проза, читаемость важнее характера.
- **Числовые данные (XP, статистика, счётчики):** табличные цифры (`font-variant-numeric: tabular-nums`), чтобы цифры в колонках не "прыгали" по ширине.
- Обязательна поддержка кириллицы (рус, каракалп.) и латиницы (узб, англ) без "квадратиков" — проверить шрифт на полное покрытие используемых алфавитов.

### Форма/скругления/тени

- Карточки — умеренное скругление (8–12px), не резко-острые углы (не newspaper-style) и не чрезмерно "мягкие" (не игрушечный вид) — баланс между официальностью (админ) и дружелюбностью (ученик).
- Тени — лёгкие, только для отделения интерактивных карточек (книги, достижения) от фона; в таблицах/списках — без теней, только границы.
- Достижения (значки) — CSS 3D-эффект: лёгкий `box-shadow` снизу + `background: radial-gradient` для имитации объёма медали/значка, без хранения растровых изображений.

## 2. Адаptivность (breakpoints)

| Диапазон | Устройство | Поведение |
|---|---|---|
| `< 640px` | Смартфон | Одна колонка, нижняя панель навигации (tab bar) вместо бокового меню, таблицы → карточки (каждая строка таблицы превращается в карточку с ключ-значение парами) |
| `640–1024px` | Планшет | Две колонки где уместно, боковое меню сворачивается в иконки, таблицы остаются таблицами, но с горизонтальным скроллом при необходимости |
| `> 1024px` | Компьютер | Полное боковое меню с подписями, многоколоночные дашборды, таблицы без ограничений |

**Критично:** это одна вёрстка с CSS media queries / Tailwind breakpoints, НЕ три отдельных шаблона под устройства. Каждый экран проектируется сразу с учётом всех трёх диапазонов, а не адаптируется постфактум.

## 3. Навигация по ролям

### Суперадмин
- Районы → Школы (создание, список, поиск)
- Рейтинг школ (общий + по клику — краткий внутришкольный лидерборд)
- Сброс паролей школьных админов
- Общая статистика/графики (активность районов/школ, Chart.js)

### Школьный админ (библиотекарь)
- Дашборд (инвентаризация: книги/учебники всего/в наличии, активность чтения)
- Ученики (список, класс, профиль, сброс пароля, аварийный возврат)
- Учителя (список, профиль, предмет, сброс пароля)
- Классы (список, комплект учебников класса, назначение классных руководителей)
- Каталог (учебники — общий справочник + свой остаток; обычные книги — свои)
- Выдача/приём учебников (комплект, поштучно и массово)
- QR-сканер (для выдачи/возврата обычных книг и краткосрочной выдачи учителям)
- Новости (публикация для своей школы)
- Экспорт (шаблон импорта, список к сдаче учебников — май, выбор полный/должники)
- Переводы учеников/учителей (инициировать уход/приём)

### Учитель (обычный)
- Каталог книг/учебников своего предмета (фильтр по умолчанию, с возможностью снять)
- "Моя корзина" → бронирование одним QR
- "На руках" → выборочный возврат через QR
- История (что брал, когда — переезжает между школами)
- Общешкольный лидерборд (просмотр, без участия)
- Новости (своя школа + суперадмин)

### Учитель — классный руководитель (доп. вкладка поверх обычного учителя)
- "Мои классы" (read-only): список учеников, у кого что на руках, статус учебников
- Рейтинг своих классов

### Ученик (5–11 класс)
- Каталог обычных книг (поиск, фильтр по категории)
- "Моя корзина" → бронирование одним QR
- "На руках" → выборочный возврат через QR
- Профиль: XP, уровень, зал достижений (открытые + приглушённые заблокированные), стрик
- Недельный челлендж (тест: 15 вопросов, таймер 15 минут на весь тест)
- Лидерборд школы (по XP) + лидерборд челленджа (по классу)
- Новости (своя школа)
- Смена пароля

### Ученик 1–4 класс
- Не имеет входа в систему — раздел не существует.

## 4. Ключевые экраны — детализация поведения

### 4.1 Каталог + "Корзина" (ученик/учитель)

- Список/поиск книг — карточки с обложкой, названием, автором, статусом "есть в наличии/нет".
- Кнопка "Добавить в бронирование" на каждой карточке → появляется плавающий индикатор "Корзина (N)" (закреплён снизу на мобильном, сбоку на десктопе).
- Экран корзины — список выбранного с возможностью убрать позицию, кнопка "Подтвердить бронирование".
- После подтверждения — экран с QR-кодом на весь экран (крупно, для удобного сканирования библиотекарем), обратный отсчёт до истечения токена (2 минуты), автообновление токена по истечении без перезагрузки страницы (HTMX polling).

### 4.2 "На руках" + выборочный возврат

- Список текущих выданных позиций (книги + учебники, если учитель) с датой выдачи и due date (если есть), визуальным выделением просроченных (`--color-danger`).
- Чекбоксы для выбора, что возвращается сейчас; кнопка "Вернуть выбранное" → генерация QR на подгруппу.
- Явно показывается: "осталось на руках после этого возврата: N" — чтобы пользователь понимал, что часть остаётся.

### 4.3 QR-сканер (библиотекарь)

- Полноэкранный режим камеры на мобильном/планшете библиотекаря.
- После успешного скана — мгновенное подтверждение (звук/вибрация + визуальный чек), список того, что было в этом скане (выдано/принято).
- Ошибка скана (истёк токен, чужая школа, повреждён) — явное сообщение с причиной, без технического жаргона: "Этот QR-код устарел, попросите ученика обновить его" / "Этот код принадлежит другой школе".

### 4.4 Выдача/приём учебников (комплект)

- Экран "Выдать учебники — [ФИО ученика]" — список предметов класса построчно, по каждому: статус (не выдан+в наличии / не выдан+нет в наличии / уже выдан+дата / просрочено), чекбокс, кнопка "выдать все доступные" сверху.
- Аналогичный экран для приёма — чекбоксы "сдал".
- Массовая версия — тот же интерфейс, но список учеников класса вместо списка предметов, с возможностью развернуть каждого.

### 4.5 Зал достижений (ученик)

- Сетка значков (иконки/emoji + CSS 3D). Открытые — насыщенный цвет, дата получения при наведении/тапе. Не открытые — приглушённый серый силуэт с подсказкой условия ("Прочитай ещё 3 книги").
- Прогресс-бар к следующему уровню (XP до следующего уровня).
- Виджет текущего стрика (иконка огня + число дней), с индикатором "заморозки", если есть.
- В чужом профиле/лидерборде — только ряд полученных значков без деталей и без незавершённых.

### 4.6 Недельный челлендж (тест)

- Стартовый экран: "У вас 15 минут на 15 вопросов" + кнопка "Начать".
- Экран вопроса: номер вопроса (не общий таймер на вопрос — единый таймер сверху на весь тест, всегда видимый), 3 варианта ответа как крупные тап-зоны (удобно с телефона).
- Общий таймер — заметный, но не паникующий (меняет цвет на `--color-warning` за 2 минуты до конца, `--color-danger` за 30 секунд).
- При перезагрузке страницы — восстановление на том же вопросе с корректным оставшимся временем (запрос состояния к серверу, не локальный JS-таймер).
- Экран результата — количество правильных, полученный XP, место в лидерборде класса.

### 4.7 Лидерборды

- Список: место, имя (сокращённое ФИО для приватности — например, "Иванов И." или по решению школы), XP/очки, значки последних 2-3 достижений.
- Своя строка визуально выделена (текущий пользователь всегда видит себя, даже если он не в топ-10 — закреплена внизу списка отдельно, если не входит в видимый диапазон).

### 4.8 Инвентаризация/статистика (админ школы, суперадмин)

- Карточки с крупными цифрами сверху (всего книг, всего учебников, активных читателей) — использование Display-шрифта для чисел.
- Графики (Chart.js) — динамика по месяцам, распределение по категориям — ниже карточек.
- Таблица должников/просрочек — с быстрым фильтром и кнопкой экспорта в Excel.

## 5. Near-realtime обновления (HTMX polling)

- Компоненты, требующие обновления без перезагрузки: статус QR-токена (обратный отсчёт + автообновление), статус скана (у библиотекаря), счётчик "Корзина (N)", уведомления (колокольчик).
- Реализация — `hx-trigger="every 3s"` (или аналогичный интервал) на соответствующих фрагментах, подгружающих обновлённый HTML-кусок с сервера.
- Полноценные WebSockets (Django Channels) — не в первой версии, оставить архитектурную возможность подключить позже для конкретных экранов (например, живой QR-скан в реальном времени между двумя открытыми окнами), если станет критично, и хостинг это выдержит.

## 6. Уведомления (колокольчик + push + баннер)

Раз в разговоре прозвучало "нужно современнее" без уточнения деталей — предлагаемый вариант, требующий проверки с человеком отдельно:
- Иконка колокольчика в шапке с числом непрочитанных, дропдаун со списком последних уведомлений (не отдельная страница по умолчанию).
- Верхний баннер — только для по-настоящему важных/срочных уведомлений (не рутинных), закрываемый, не более одного одновременно.
- Push-уведомления — через Web Push API (браузерные, бесплатно, не требует стороннего платного сервиса) — для событий вроде "у вас просрочен учебник" или "новая школьная новость".

## 7. Страницы ошибок

- Кастомные шаблоны 404 / 403 / 500 / 400 в общем визуальном стиле (не дефолтный Django-текст).
- На языке текущего пользователя (через i18n, как весь остальной интерфейс).
- Тон — по принципу "без извинений, чётко что произошло и что делать": например, 403 — "У вас нет доступа к этой странице. Если это ошибка, обратитесь к администратору школы." Не "Упс! Что-то пошло не так 🙈".
- 404 — с кнопкой возврата на дашборд соответствующей роли (не на общую главную).

## 8. i18n во фронтенде

- Переключатель языка — в шапке, доступен всегда, применяется мгновенно (без перезагрузки, если технически реализуемо через HTMX, иначе — с перезагрузкой текущей страницы).
- Язык интерфейса пользователя (`user.language`) — личная настройка, независима от языка класса (`class.language`), который используется только для Excel-экспорта списка к сдаче учебников (см. основной документ, раздел 13).
- Направление текста — все 4 языка (узб, рус, каракалп, англ) используют латиницу/кириллицу слева-направо, RTL не требуется.

## 9. Состояния пустых экранов и загрузки

- Пустой каталог/список ("Пока нет книг в этой категории") — с понятным следующим действием, а не голая надпись.
- Загрузка данных (HTMX) — скелетоны (skeleton loaders) для таблиц/карточек вместо спиннера на весь экран, особенно для polling-обновлений (не должно "мигать" на каждое обновление).
- Загрузка QR-сканера — явный индикатор "Наведите камеру на QR-код" вместо пустого чёрного экрана.

## 10. Открытые вопросы для уточнения перед реализацией

- Точный шрифт для display/body (нужен доступ проверить лицензии/доступность через Google Fonts или аналог).
- Формат отображения имени в лидерборде — полное ФИО, сокращённое, или по выбору школы/ученика (вопрос приватности).
- Нужны ли анимации перехода между экранами геймификации (открытие достижения, левелап) — и насколько выраженные, чтобы не выглядело избыточно.
- Точное поведение push-уведомлений — какие события триггерят push, а какие только in-app.
